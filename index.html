WHAT THIS TOY IS TEACHING

Rules are always-on. The real system keeps a small rules file that every AI reads before every task — protect the human, never break the live show, back up first, nothing publishes without approval. Try running the "lights" scenario ON AIR with the rules slot empty, then again with the show rule loaded. Same request, opposite outcome.

Memory is searched, not memorized. There are hundreds of little memory files — taste ("he hates cheesy effects"), scar tissue ("light #4 won't turn off"), gotchas. The AI doesn't read them all; it searches for the relevant ones. And when a run teaches it something, it writes a new memory — that's the green wire looping back. The system literally gets smarter every day it's used.

Many brains, one board. The real version runs several AI chats at once, working in parallel lanes. They coordinate through a shared board file — who owns what, what's blocked, what only the human can decide. Decisions the human makes get written down once and every brain obeys them forever after.

Verify, don't trust. The deepest rule in the set: after any change, look at the real output — the actual pixels, the actual frames — not the settings screen. Run the "lag" scenario with and without the verify rule to feel why.
